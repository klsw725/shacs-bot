테스트 프레임워크 없음 (AGENTS.md 명시). 코드 리뷰 + 정적 분석으로 검증.

## 검증 결과

### 검증 1: researcher 역할 spawn → write_file/edit_file 미등록

**코드 경로**: `_run_subagent()` → `all_tools` 7개 생성 → `role_config.allowed_tools` 기반 필터링

```python
# subagent.py:208-210
for tool in all_tools:
    if not role_config.allowed_tools or tool.name in role_config.allowed_tools:
        tools.register(tool)
```

- researcher `allowed_tools = ["read_file", "list_dir", "exec", "web_search", "web_fetch"]`
- `not allowed_tools` → `False` (non-empty)
- `write_file`, `edit_file` → `tool.name in allowed_tools` → `False`
- **결과: 미등록 확인** ✅

### 검증 2: executor 역할 spawn → 전체 도구 등록

- executor `allowed_tools = []`
- `not []` → `True` → 모든 도구 무조건 등록
- **결과: 7개 전체 등록 확인** ✅

### 검증 3: role 미지정 → executor 기본 동작

기본값 체인 (3단계 모두 일관):
1. `SpawnTool.execute(role: str = "executor")`
2. `SubagentManager.spawn(role: str = "executor")`
3. `SubagentManager._run_subagent(role: str = "executor")`
4. 안전장치: `SUBAGENT_ROLES.get(role, SUBAGENT_ROLES["executor"])` — 알 수 없는 역할도 executor 폴백
- **결과: 하위 호환 유지 확인** ✅
