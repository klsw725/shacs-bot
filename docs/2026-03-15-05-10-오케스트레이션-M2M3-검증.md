# 오케스트레이션 시스템 프롬프트 M2+M3: 시나리오 검증

> **Prompt**: `D Next` → orchestration-prompt PRD의 M2+M3를 통합 검증

## 검증 방법

`uv run shacs-bot agent -m "..." --logs` 싱글 메시지 모드로 테스트. 모델: `openai-codex/gpt-5.3-codex`

## M2 검증 결과: 기본 대화 시나리오

### 테스트 1: 단순 인사 ("안녕")
- **결과**: 도구 호출 없이 직접 답변 ✅
- **응답**: "안녕하세요! 무엇을 도와드릴까요?"
- **불필요한 서문 없음** ✅

### 테스트 2: 감사 표현 ("고마워")
- **결과**: 도구 호출 없이 직접 답변 ✅
- **응답**: "천만에요! 필요하면 언제든 불러주세요."

### 테스트 3: 단순 질문 ("파이썬에서 리스트를 역순으로 정렬하는 방법은?")
- **결과**: 도구 호출 없이 직접 답변 ✅
- **응답**: 3가지 방법 (reverse, 슬라이싱, sort) 코드 예제 포함 구조적 답변

### 테스트 4: 파일 읽기 요청 ("워크스페이스의 SOUL.md 파일을 읽어줘")
- **결과**: `read_file` 도구 직접 사용 ✅
- **파일 미발견 시**: graceful 처리 — 원인 설명 + 대안 제안 ✅

### 테스트 5: 웹 검색 필요 요청 ("오늘 한국 주요 뉴스를 웹에서 검색해서...")
- **결과**: `web_search` 도구 3회 호출 시도 (직접 처리)
- **BRAVE_API_KEY 미설정으로 실패** → 원인 정확히 안내 + 해결 방법 제시 ✅
- **도구 실패 복구 동작 확인** ✅

## M3 검증 결과: 서브에이전트 위임 시나리오

### 테스트 6: 조사 요청 ("2025년 AI 에이전트 프레임워크 트렌드를 조사해줘")
- **결과**: 서브에이전트 미사용, 직접 답변
- **분석**: 모델이 이미 알고 있는 내용이라 직접 처리. SOUL.md는 서브에이전트 "활용 고려"로 표현 — 필수 아님. **합리적 판단** ✅

### 테스트 7: 백그라운드 파일 작업 ("백그라운드에서 워크스페이스의 모든 .md 파일 목록을 정리해줘")
- **결과**: `spawn(role="executor")` 사용 ✅
- **서브에이전트**: `exec` 도구로 `find` 명령 실행
- **역할 선택**: executor — 파일 작업이므로 정확 ✅

### 테스트 8: 백그라운드 분석 요청 ("백그라운드에서 AGENTS.md 파일을 분석해서 핵심 내용을 요약해줘")
- **결과**: `spawn(role="executor")` 사용 — analyst가 더 적절
- **분석**: 파일을 찾고 읽는 작업이 포함되어 모델이 executor 선택. 순수 분석만 필요한 경우 analyst 역할이 더 적합.
- **M4 튜닝 대상**: SOUL.md에서 analyst 역할 선택 기준을 더 명확히 안내 필요

## M2 성공 기준 평가

| 기준 | 결과 |
|---|---|
| 단순 대화에 도구 호출 없이 직접 답변 | ✅ |
| 응답에 불필요한 서문 없음 | ✅ |
| 도구 실패 시 다른 접근 또는 상황 설명 | ✅ |
| 파일 작업 요청에 적절한 도구 사용 | ✅ |

## M3 성공 기준 평가

| 기준 | 결과 |
|---|---|
| 백그라운드 작업 요청에 spawn 사용 | ✅ |
| 파일 작업에 executor 역할 선택 | ✅ |
| 조사 요청에 researcher 역할 선택 | ⚠️ 직접 처리 (합리적이나 개선 여지) |
| 분석 요청에 analyst 역할 선택 | ⚠️ executor 선택 (M4 튜닝 대상) |

## M4 튜닝 시 개선 사항

1. **analyst 역할 선택 빈도**: SOUL.md에서 "분석/요약" 요청 시 analyst 역할을 더 적극적으로 선택하도록 가이드 강화
2. **researcher 역할 선택**: 웹 검색이 필요한 조사에서 researcher 서브에이전트 활용 빈도 개선
3. **BRAVE_API_KEY 설정**: 웹 검색 기능 완전 테스트를 위해 API 키 설정 필요

## 환경 제약

- BRAVE_API_KEY 미설정: web_search 도구 사용 불가 → 웹 조사 시나리오 완전 검증 불가
- cron/heartbeat 퇴행 테스트: 별도 게이트웨이 모드 실행 필요 (이번 검증 범위 외)
