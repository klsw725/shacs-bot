# config.json 글로벌 환경변수 주입

> **프롬프트**: `NOTION_KEY 같은 환경변수를 주입하고 싶은데` → PRD 작성 후 구현

## 변경 내용

config.json 최상위에 `env` 필드를 추가하여, 프로세스 시작 시 `os.environ`에 주입하는 기능.

```json
{
  "env": {
    "NOTION_KEY": "ntn_xxx",
    "BRAVE_API_KEY": "BSA_xxx"
  }
}
```

## 변경 파일

| 파일 | 변경 |
|---|---|
| `shacs_bot/config/schema.py` | `Config`에 `env: dict[str, str]` 필드 추가 |
| `shacs_bot/config/loader.py` | `_apply_env()` 함수 추가, `load_config()`에서 호출 |
| `docs/prds/global-env-injection.md` | PRD 작성 |

## 동작 규칙

- `os.environ.setdefault` 사용 — 시스템 환경변수가 이미 있으면 config 값 무시
- 빈 문자열 value는 주입하지 않음
- provider `_setup_env`보다 먼저 실행됨
- 하위 호환: `env` 미지정 시 기존과 동일

## 검증

- Config 기본값 (빈 dict) 확인
- env 값 → os.environ 주입 확인
- 빈 문자열 스킵 확인
- setdefault 동작 (기존 값 보존) 확인
- LSP 에러: 내 변경으로 인한 새 에러 없음
