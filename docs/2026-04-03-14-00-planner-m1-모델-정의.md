## 변경 사항

### 신규 파일: `shacs_bot/agent/planner.py`

PRD M1 범위에 해당하는 플래너 데이터 모델과 step taxonomy를 정의한 새 모듈.

#### 정의된 타입

| 이름 | 종류 | 설명 |
|------|------|------|
| `StepKind` | `Literal` | 단일 step의 종류: `ask_user`, `research`, `summarize`, `wait_until`, `request_approval`, `send_result` |
| `PlanKind` | `Literal` | 플래너 라우팅 결정: `direct_answer`, `clarification`, `planned_workflow` |
| `PlanStep` | Pydantic 모델 | step 하나 — kind, description, depends_on(인덱스 목록), notify 포함 |
| `AssistantPlan` | Pydantic 모델 | 전체 처리 계획 — kind, steps, clarification_question, summary 포함 |
| `ClarificationResult` | Pydantic 모델 | clarification 해소 결과 — question, answer, original_request 포함 |

#### 설계 결정

- 모든 모델은 `frozen=True` (`ConfigDict`)로 불변 처리 — 계획 객체의 의도치 않은 변경 방지.
- camelCase alias 없음 — 외부 JSON 직렬화 대상이 아닌 내부 Python 타입.
- `Optional[X]` / `Union[X, Y]` 대신 `X | None` 사용 (AGENTS.md 규칙 준수).
- `print()` 없음, `loguru` 미사용 (순수 모델 정의 파일).

#### M2/M3 연계 지점

- M2 (`loop.py` 분기): `AssistantPlan.kind`로 `direct_answer` / `clarification` / `planned_workflow` 경로 분기.
- M3 (session metadata): `AssistantPlan`을 `Session.metadata["current_plan"]`에 저장.
- M3 (workflow handoff): `planned_workflow` 계획을 `WorkflowRuntime`으로 전달 시 `PlanStep` 목록 사용.
