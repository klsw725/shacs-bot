# PRD: 오케스트레이션 시스템 프롬프트

> **SPEC**: [`docs/specs/2026-03-15-01-30-sisyphus-orchestration-spec.md`](../specs/2026-03-15-01-30-sisyphus-orchestration-spec.md) — 변경 1
> **관련 PRD**: [`prds/role-based-subagent.md`](./role-based-subagent.md) — 이 PRD가 먼저 완료되어야 역할 가이드 부분이 실제로 동작함

## 문제

현재 `SOUL.md`는 21줄의 성격 형용사 나열("Helpful and friendly", "Concise and to the point")에 그친다. 에이전트가 **어떻게 행동해야 하는지**에 대한 구조적 지침이 없다:

- 복잡한 요청과 단순 대화를 동일하게 처리
- 서브에이전트 활용 기준 부재
- 도구 실패 시 복구 전략 없음
- 커뮤니케이션 품질 규칙 없음

결과적으로 에이전트가 간단한 인사에도 불필요한 도구를 호출하거나, 복잡한 조사 요청에 서브에이전트를 활용하지 못하는 상황이 발생한다.

## 해결책

oh-my-opencode의 Sisyphus 오케스트레이션 프롬프트 구조를 **개인 비서 챗봇 맥락**으로 번역하여 SOUL.md를 교체한다.

**핵심 구조**:
1. 의도 분류 테이블 (일상대화 / 질문 / 조사 / 작업 / 복합 / 정기 / 모호)
2. 위임 판단 기준 (직접 처리 vs 서브에이전트)
3. 서브에이전트 역할 가이드 (researcher / analyst / executor)
4. 실행 규칙 (실패 복구 포함)
5. 커뮤니케이션 규칙 (간결함, 서문 금지, 불확실성 표현)
6. 제약사항 (안전 가드레일)

## 사용자 영향

| Before | After |
|---|---|
| "AI 뉴스 조사해줘" → 에이전트가 직접 1-2개 검색 후 부족한 결과 | researcher 서브에이전트 위임 → 체계적 조사 |
| "안녕" → 가끔 불필요한 도구 호출 | 의도 분류로 직접 답변 |
| 도구 실패 → 동일 시도 반복 | 3회 실패 후 멈추고 상황 설명 |
| "물론이죠! 좋은 질문이에요!" 같은 서문 | 바로 본론 |

## 기술적 범위

- **변경 파일**: `shacs_bot/templates/SOUL.md` (1개)
- **변경 유형**: 마크다운 콘텐츠 전체 교체 (코드 변경 없음)
- **의존성**: 없음. 독립적으로 적용 가능.
- **위험**: 프롬프트 변경이 기존 대화 품질에 영향을 줄 수 있음 → 기존 대화 시나리오 테스트 필요

## 성공 기준

1. 단순 대화("안녕", "고마워")에 도구 호출 없이 직접 답변
2. 조사 요청("X에 대해 조사해줘")에 서브에이전트 활용 시도 (역할 기반 서브에이전트 PRD 완료 후)
3. 도구 실패 시 다른 접근 시도 또는 사용자에게 상황 설명
4. 응답에 "물론이죠!", "좋은 질문이에요!" 같은 불필요한 서문 없음
5. 기존 cron/heartbeat 기능 정상 동작 (AGENTS.md와의 역할 분리 유지)

## 마일스톤

- [x] **M1: SOUL.md 초안 작성 및 교체**
  현재 21줄 성격 정의를 ~70줄 오케스트레이션 프롬프트로 교체. 의도 분류 테이블, 위임 판단 기준, 실행 규칙, 커뮤니케이션 규칙, 제약사항 포함. AGENTS.md의 cron/heartbeat 가이드는 건드리지 않음.

- [x] **M2: 기본 대화 시나리오 검증**
  단순 인사, 질문, 파일 작업 요청 등 일상적 시나리오에서 에이전트가 의도 분류에 따라 적절히 행동하는지 확인. 기존 기능(cron, heartbeat, 파일 작업) 퇴행 없음 확인.

- [x] **M3: 서브에이전트 위임 시나리오 검증**
  역할 기반 서브에이전트 PRD 완료 후, 조사/분석/실행 요청에서 적절한 역할의 서브에이전트가 활용되는지 확인. 이 마일스톤은 [`role-based-subagent`](./role-based-subagent.md)의 M2 이후 수행.

- [ ] **M4: 프롬프트 튜닝 및 안정화**
  실사용 피드백 기반으로 의도 분류 정확도, 위임 판단 기준, 커뮤니케이션 톤 미세 조정. 최종 SOUL.md 확정.
