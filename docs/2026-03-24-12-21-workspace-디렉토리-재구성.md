# Workspace 디렉토리 재구성

> **Prompt**: `D Next` → PRD 분석 후 M1~M5 전체 구현
> **PRD**: `prds/workspace-reorganization.md`
> **Spec**: `docs/specs/2026-03-24-workspace-reorganization.md`

## 변경 요약

`~/.shacs-bot/` 디렉토리를 역할 기반으로 분리. 시스템 전용 런타임 데이터를 `data/` 하위로 이동하고, 스킬 출력물을 `workspace/sandbox/`로 정리.

## 변경 파일

| 파일 | 변경 |
|------|------|
| `config/paths.py` | 6개 경로 헬퍼 추가 (`get_data_subdir`, `get_sessions_dir`, `get_clawhub_dir`, `get_media_downloads_dir`, `get_sandbox_dir`, `get_agents_dir`), `get_cron_dir`/`get_usage_dir` 내부 구현 변경 |
| `config/loader.py` | `_migrate_workspace_layout()` 추가 (6개 이전 경로, 멱등), `load_config()` 내에서 자동 호출 |
| `agent/session/manager.py` | `workspace / "sessions"` → `get_sessions_dir()` |
| `cli/commands.py` | gateway 크론 경로 `config.workspace_path / "cron"` → `get_cron_dir()` (CLI와 통일) |
| `channels/telegram.py` | `get_media_dir()` → `get_media_downloads_dir("telegram")` |
| `channels/feishu.py` | `get_media_dir()` → `get_media_downloads_dir("feishu")` |
| `channels/discord.py` | `get_media_dir()` → `get_media_downloads_dir("discord")` |
| `channels/matrix.py` | `get_media_dir("matrix")` → `get_media_downloads_dir("matrix")` |
| `templates/TOOLS.md` | sandbox 출력 가이드라인 추가 |
| `skills/summarize/SKILL.md` | `sandbox/summarize/` 파일 출력 규칙 안내 |

## 새 디렉토리 구조

```
~/.shacs-bot/
├── workspace/          # LLM 접근 데이터
│   ├── sandbox/        # 스킬/에이전트 출력물 (신규)
│   ├── memory/         # 장기 메모리 (변경 없음)
│   └── media/          # 생성 미디어 (변경 없음)
├── data/               # 시스템 전용 런타임 (신규)
│   ├── sessions/
│   ├── cron/
│   ├── usage/
│   └── clawhub/
├── media/              # 채널 다운로드 (채널별 하위 폴더)
│   └── {channel}/
└── agents/             # 에이전트 스토어 (신규)
```

## 마이그레이션 동작

앱 시작 시 `load_config()` 직후 자동 실행. 멱등 (이미 이전된 경우 무시).

이전 경로:
- `workspace/sessions/` → `data/sessions/`
- `workspace/.clawhub/` → `data/clawhub/`
- `workspace/cron/` → `data/cron/`
- `~/.shacs-bot/cron/` → `data/cron/` (상위 cron 통합)
- `~/.shacs-bot/usage/` → `data/usage/`
- `~/.shacs-bot/sessions/` → `data/sessions/` (레거시)
