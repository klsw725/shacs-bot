# LLM 플래너 폴백 step_meta 강화

**날짜**: 2026-04-03  
**브랜치**: feature/planned-workflow-executor

## 사용자 프롬프트

> Implement the smallest safe improvement so the hybrid LLM planner fallback produces and preserves better structured `step_meta` for executor-facing steps. Strengthen the `_LLM_PLANNER_SYSTEM` contract so `wait_until` prefers `iso_time` or `duration_minutes`, and `ask_user` / `request_approval` prefer `prompt` in `step_meta`. Preserve backward compatibility for fallback outputs that still omit metadata. If metadata can be derived cheaply after parse (for example from wait_until description), normalize it there rather than failing. Add/update deterministic smoke tests using stub provider output to verify metadata creation/normalization.

## 변경 파일

### `shacs_bot/agent/loop.py`

1. **`_LLM_PLANNER_SYSTEM` 프롬프트 강화** (`step_meta` 계약 명시)  
   - `wait_until` 단계에 `iso_time` (ISO 8601) 또는 `duration_minutes` (정수) 포함 지시
   - `ask_user` 단계에 `prompt` 키 포함 지시  
   - `request_approval` 단계에 `prompt` 키 포함 지시
   - JSON 스키마 예시 포함

2. **`_normalize_llm_plan_metadata` 정적 메서드 추가**  
   LLM이 `step_meta`를 생략했을 때 description에서 도출 가능한 메타데이터를 보충:
   - `wait_until`: `iso_time`/`duration_minutes` 모두 없으면 `parse_wait_until_time(description)` 호출해 `iso_time` 채움
   - `ask_user` / `request_approval`: `prompt` 없으면 `description` 값으로 채움
   - 기존 `step_meta`가 올바른 키를 이미 가지면 변경 없음 (보존 우선)

3. **`_llm_classify_fallback`에서 정규화 호출**  
   `AssistantPlan.model_validate(data)` 이후 `self._normalize_llm_plan_metadata(plan)` 호출.

### `scripts/smoke_llm_planner_fallback.py`

검증 13~17 추가:
- **검증 13**: LLM이 `wait_until` + `iso_time` step_meta 반환 → 기존 값 보존
- **검증 14**: LLM이 `wait_until` step_meta 없이 반환 → description에서 `iso_time` 정규화
- **검증 15**: LLM이 `ask_user` + `prompt` step_meta 반환 → 기존 값 보존
- **검증 16**: LLM이 `ask_user` step_meta 없이 반환 → `description`을 `prompt`로 정규화
- **검증 17**: LLM이 `request_approval` step_meta 없이 반환 → `description`을 `prompt`로 정규화

## 설계 결정

- **description 폴백 제거 안 함**: executor는 `step_meta.prompt`가 없으면 `description`을 사용하는 기존 경로를 유지. 정규화는 그 경로보다 앞단에서 메타데이터를 채우는 것이므로 하위 호환성 보장.
- **신규 의존성 없음**: `parse_wait_until_time`은 이미 `loop.py`에서 임포트 중.
- **아키텍처 변경 없음**: `_llm_classify_fallback` 반환 타입, 호출자 시그니처 모두 유지.
