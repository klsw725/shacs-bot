# Vector Memory M1 구현

## 사용자 프롬프트

> 브랜치 만들고 거기서 진행

## 작업 개요

Vector Memory PRD의 M1 범위만 반영했다. 새 브랜치 `feature/vector-memory-m1`에서 optional dependency, 설정, 벡터 메모리 모듈, `search_history` hybrid 검색 경로, AgentLoop/Subagent/CLI 배선을 추가했다.

## 변경 사항

### 1. optional dependency 추가
- `pyproject.toml`
  - `vector-memory` extra 추가
  - `lancedb`, `sentence-transformers`를 선택 설치 의존성으로 등록

### 2. 설정 추가
- `shacs_bot/config/schema.py`
  - `VectorMemoryConfig` 추가
  - `ToolsConfig.vector_memory` 추가

### 3. 벡터 메모리 모듈 추가
- `shacs_bot/memory/vector.py`
  - LanceDB + sentence-transformers가 설치된 경우에만 초기화되는 `VectorMemory` 구현
  - `initialize`, `add`, `search`, `is_available` 제공
- `shacs_bot/memory/__init__.py`
  - 패키지 경로 추가

### 4. 기존 메모리/도구 통합
- `shacs_bot/agent/memory.py`
  - `MemoryStore`에 vector config 입력 추가
  - `append_history()` 시 벡터 인덱스에도 기록
  - `semantic_search()` 추가
  - `MemoryConsolidator`가 vector-enabled `MemoryStore`를 쓰도록 변경
- `shacs_bot/agent/tools/history.py`
  - `mode=grep|semantic|hybrid` 파라미터 추가
  - grep 결과와 semantic 결과를 dedupe 후 함께 반환
- `shacs_bot/agent/tools/registry.py`
  - 기본 도구 생성 시 vector-enabled `MemoryStore`를 `SearchHistoryTool`에 주입

### 5. 런타임 배선
- `shacs_bot/agent/loop.py`
- `shacs_bot/agent/subagent.py`
- `shacs_bot/cli/commands.py`
  - `config.tools.vector_memory`가 메인 루프와 서브에이전트 기본 도구 생성까지 전달되도록 연결

## 검증

### 실행 검증
- `uv run python -m compileall shacs_bot`
  - 전체 Python 모듈 컴파일 성공
- `uv run python -c "...SearchHistoryTool smoke..."`
  - `mode=hybrid`에서 grep fallback 경로 정상 동작 확인

### 진단 메모
- 새 파일 `shacs_bot/memory/vector.py`는 LSP 기준 **error 없음** 상태로 정리했다.
- 일부 기존 파일(`agent/memory.py`, `cli/commands.py`, `config/schema.py` 등)은 이번 변경 전부터 basedpyright 경고/에러가 남아 있었다.
- 이번 작업은 그 기존 타입 정리 작업까지 확장하지 않고, Vector Memory M1 범위만 외과적으로 반영했다.

## 비고

- `uv.lock`은 `pyproject.toml`의 optional dependency 변경에 따라 함께 갱신된 상태다.
- Vector dependency가 설치되지 않은 환경에서는 기존 grep 검색만 유지된다.
