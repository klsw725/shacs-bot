# 역할 선택 프롬프트 튜닝 (오케스트레이션 M4 + 역할기반 서브에이전트 M4)

> **Prompt**: `D Next` → 두 PRD의 M4 통합 작업 추천 → 진행

## 문제

M3 검증에서 발견된 역할 선택 오류:
1. "AGENTS.md 파일을 분석해서 요약해줘" → `executor` 선택 (analyst가 적절)
2. "AI 트렌드 조사해줘" → 직접 처리 (합리적이나 웹 검색 미활용)

## 변경 파일

| 파일 | 변경 |
|---|---|
| `shacs_bot/templates/SOUL.md` | 의도 파악 테이블에 "분석/요약" 유형 추가, 위임 판단에서 "긴 문서" 조건 제거 및 "작업의 목적으로 역할을 선택한다" 원칙 추가 |
| `shacs_bot/agent/tools/spawn.py` | role 파라미터 description 강화 — "작업 목적에 맞는 역할을 반드시 지정하세요" 원칙 + 각 역할별 사용 조건 명확화 |
| `docs/prds/orchestration-prompt.md` | M4 완료 표시 + 진행 로그 |
| `docs/prds/role-based-subagent.md` | M4 완료 표시 + 진행 로그 |

## 핵심 변경 내용

### SOUL.md 의도 파악 테이블

**Before**: "정보 조사" 유형에 "비교해줘" 포함, "분석/요약" 유형 없음
**After**: "분석/요약" 유형 분리 추가 ("분석해줘", "요약해줘", "비교해줘" → analyst 위임 고려)

### SOUL.md 위임 판단

**Before**: "긴 문서를 읽고 분석/요약해야 하는 작업 → analyst" (길이 조건이 analyst 선택 방해)
**After**: "파일/데이터를 분석하거나 요약하는 작업 → analyst" (작업 목적 기준으로 변경)

### spawn.py role description

**Before**: "서브에이전트 역할. researcher: ... analyst: ... executor: ... (기본값)"
**After**: "작업 목적에 맞는 역할을 반드시 지정하세요. analyst: 파일/데이터를 분석하거나 요약할 때... researcher: 웹에서 정보를 찾거나 조사할 때... executor: 파일을 생성/수정하거나 명령을 실행할 때."

## 발견 사항

- SOUL.md 변경만으로는 불충분. 모델(gpt-5.3-codex)이 tool description을 SOUL.md보다 강하게 참조
- spawn tool의 role description과 SOUL.md 가이드를 함께 튜닝해야 효과적
- 모델이 이미 아는 내용("2025년 AI 트렌드")은 직접 처리 — 합리적 판단. 최신 정보("2026년 3월 뉴스")만 researcher 위임
