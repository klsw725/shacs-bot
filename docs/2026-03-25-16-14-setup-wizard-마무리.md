# Interactive Setup Wizard 마무리

## 프롬프트

> 일단 코드를 보고 저게 완료되었는지 다시 파악해보고 완료 되었다면 문서에 표시해. 미 완료로 보인다면 나에게 알려줘
> B인데 이건 main branch에서 진행해주라

## 변경 요약

`main` 브랜치에서 Interactive Setup Wizard의 남은 갭 3개를 정리했다.

- Tools 메뉴 추가 (`web`, `exec`, `media`)
- `Exit Without Saving`가 실제로 저장하지 않도록 수정
- PRD 설명을 shipped behavior에 맞게 `context_window` 대신 모델 context 기반 `max_tokens` 추천으로 정리

## 변경 파일

| 파일 | 변경 유형 | 설명 |
|------|----------|------|
| `shacs_bot/cli/onboard.py` | 수정 | Tools 메뉴 추가, discard 시 `None` 반환, summary에 tool 상태 표시 |
| `shacs_bot/cli/commands.py` | 수정 | wizard에서 `None` 반환 시 `save_config()`를 호출하지 않도록 변경 |
| `docs/specs/litellm-removal/prds/litellm-removal-and-setup-wizard.md` | 수정 | Track B 완료 반영, milestone 체크, 설명을 실제 동작과 일치시킴 |

## 검증 포인트

- `onboard --wizard` 호출 경로에 `--wizard` 옵션이 유지됨
- discard 경로에서 설정 저장이 발생하지 않음
- PRD의 Track B milestone이 현재 구현 상태와 일치함

## 메모

- 이번 변경은 `context_window` 설정 필드를 새로 도입하지 않았다.
- 현재 스키마에는 해당 필드가 없으므로, wizard는 모델 context를 기준으로 `max_tokens` 추천값을 제시하는 방식으로 문서를 정리했다.
