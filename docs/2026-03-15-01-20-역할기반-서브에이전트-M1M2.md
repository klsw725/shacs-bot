# 역할 기반 서브에이전트 M1+M2 구현

> **Prompt**: `D Next` → role-based-subagent PRD의 M1+M2를 최우선 작업으로 추천받아 구현

## 변경 파일

| 파일 | 변경 |
|---|---|
| `shacs_bot/agent/subagent.py` | `SubagentRole` dataclass, 3개 역할 프롬프트, `SUBAGENT_ROLES` 딕셔너리 추가. `spawn()`, `_run_subagent()`, `_build_subagent_prompt()`에 `role` 파라미터 통합. |
| `shacs_bot/agent/tools/spawn.py` | `parameters`에 `role` 필드 추가. `execute()`에 `role` 파라미터 추가. |
| `docs/prds/role-based-subagent.md` | M1, M2 체크박스 완료 표시. 진행 로그 추가. |

## 핵심 변경 내용

### SubagentRole 구조 (M1)
- `@dataclass(frozen=True)` SubagentRole: `system_prompt`, `allowed_tools`, `max_iterations`
- 3개 역할 프롬프트: RESEARCHER_PROMPT (정보 수집), ANALYST_PROMPT (분석/요약), EXECUTOR_PROMPT (작업 실행)
- SUBAGENT_ROLES 딕셔너리: researcher/analyst는 읽기 전용(write_file, edit_file 제외), executor는 전체 도구

### spawn/run 통합 (M2)
- `spawn(role="executor")` 기본값으로 하위 호환성 유지
- `_run_subagent()`에서 `role_config.allowed_tools` 기반 도구 필터링
- `_build_subagent_prompt(role_config)`로 역할별 전문 프롬프트 적용
- `max_iterations`도 역할별 설정 (researcher/analyst: 10, executor: 15)

## 하위 호환성

- `spawn(task="...")` (role 없이) → 기존과 동일하게 executor로 동작
- 기존 코드 경로 100% 유지
