# 백그라운드 작업 스레드 메타데이터 유실 수정

**날짜**: 2026-03-18
**프롬프트**: Discord에서 새 채팅을 열어서 채팅하고 백그라운드 작업 들어간 다음에 새 채팅을 열고 답장을 했더니 백그라운드 답장이 엉뚱한 채팅방에서 나왔다. 스레드에 기록되어야 하는데.

## 문제

백그라운드 작업(spawn/subagent) 완료 후 응답이 원래 스레드가 아닌 상위 채널로 전송됨.
메시지 라우팅 파이프라인에서 스레드 메타데이터(`thread_id`, `thread_ts`)가 전파되지 않았음.

## 근본 원인

`_set_tool_context` → `SpawnTool` → `SubagentManager` → `_announce_result` → `_process_message(system)` 경로에서 metadata가 전달되지 않음.

유실 지점 5곳:
1. `_set_tool_context`: metadata 파라미터 없음
2. `SpawnTool.set_context`: metadata 수신 불가
3. `SubagentManager.spawn`: origin에 metadata 미포함
4. `_announce_result`: 시스템 InboundMessage에 metadata/session_key 없음
5. `_process_message(system)`: OutboundMessage에 metadata 없음, session_key도 스레드 기반 아님

## 영향 범위

Discord (`thread_id`), Slack (`thread_ts`) — 스레드 라우팅을 사용하는 모든 채널.
Cron 작업도 동일 패턴.

## 변경 내용

| 파일 | 변경 |
|---|---|
| `agent/loop.py` | `_set_tool_context`에 `metadata`, `session_key` 파라미터 추가. 시스템 메시지 핸들러에서 `session_key_override` 및 metadata 전달 |
| `agent/tools/spawn.py` | `set_context`에 `metadata`, `session_key` 추가. `execute`에서 `origin_metadata` 전달 |
| `agent/subagent.py` | `spawn`에 `origin_metadata` 추가, origin dict에 metadata/session_key 포함. `_announce_result`에서 InboundMessage에 metadata/session_key_override 전달 |
| `agent/tools/message.py` | `set_context`에 metadata 추가, OutboundMessage 생성 시 metadata merge |
| `agent/tools/cron/cron.py` | `set_context`에 metadata 추가, `_add_job`에서 metadata 전달 |
| `agent/tools/cron/types.py` | `CronPayload`에 `metadata` 필드 추가 |
| `agent/tools/cron/service.py` | `add_job`에 metadata 파라미터 추가, 직렬화/역직렬화에 metadata 포함 |
| `cli/commands.py` | `on_cron_job`에서 OutboundMessage에 metadata 포함 |
