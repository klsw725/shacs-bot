## 보안 모범 사례

### 1. API 키 관리

**중요**: API 키를 버전 관리 시스템에 절대 커밋하지 마세요.

```bash
# ✅ 올바른 방법: 제한된 권한으로 설정 파일에 저장
chmod 600 ~/.shacs-bot/config.json

# ❌ 잘못된 방법: 코드에 키를 하드코딩하거나 커밋
```

**권장 사항:**
- API 키를 `~/.shacs-bot/config.json`에 저장하고 파일 권한을 `0600`으로 설정
- 민감한 키에 대해 환경 변수 사용 고려
- 프로덕션 배포 시 OS 키링/자격 증명 관리자 사용
- API 키를 정기적으로 교체
- 개발용과 프로덕션용 API 키를 분리하여 사용

### 2. 채널 접근 제어

**중요**: 프로덕션 환경에서는 반드시 `allowFrom` 목록을 설정하세요.

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "token": "YOUR_BOT_TOKEN",
      "allowFrom": ["123456789", "987654321"]
    },
    "whatsapp": {
      "enabled": true,
      "allowFrom": ["+1234567890"]
    }
  }
}
```

**보안 참고사항:**
- `v0.1.4.post3` 이전 버전에서는 빈 `allowFrom`이 모든 사용자를 허용했습니다. `v0.1.4.post4`부터 빈 `allowFrom`은 기본적으로 모든 접근을 차단합니다 — 모든 사용자를 명시적으로 허용하려면 `["*"]`를 설정하세요.
- Telegram 사용자 ID는 `@userinfobot`에서 확인하세요
- WhatsApp의 경우 국가 코드가 포함된 전체 전화번호를 사용하세요
- 비인가 접근 시도에 대한 접근 로그를 정기적으로 검토하세요

### 3. 셸 명령어 실행

`exec` 도구는 셸 명령어를 실행할 수 있습니다. 위험한 명령어 패턴은 차단되지만, 다음 사항을 준수해야 합니다:

- ✅ 에이전트 로그에서 모든 도구 사용을 검토
- ✅ 에이전트가 실행하는 명령어를 이해
- ✅ 제한된 권한을 가진 전용 사용자 계정 사용
- ✅ shacs-bot을 루트로 실행하지 않기
- ❌ 보안 검사를 비활성화하지 않기
- ❌ 민감한 데이터가 있는 시스템에서 신중한 검토 없이 실행하지 않기

**차단되는 패턴:**
- `rm -rf /` - 루트 파일 시스템 삭제
- 포크 폭탄
- 파일 시스템 포맷 (`mkfs.*`)
- 원시 디스크 쓰기
- 기타 파괴적인 작업

### 4. 파일 시스템 접근

파일 작업에는 경로 순회 방지가 적용되어 있지만, 다음 사항을 준수하세요:

- ✅ 전용 사용자 계정으로 shacs-bot 실행
- ✅ 파일 시스템 권한으로 민감한 디렉토리 보호
- ✅ 로그에서 파일 작업을 정기적으로 감사
- ❌ 민감한 파일에 대한 무제한 접근 권한 부여 금지

### 5. 네트워크 보안

**API 호출:**
- 모든 외부 API 호출은 기본적으로 HTTPS 사용
- 요청 중단 방지를 위한 타임아웃 설정
- 필요 시 방화벽을 사용하여 아웃바운드 연결 제한 고려

**WhatsApp 브리지:**
- 브리지는 `127.0.0.1:3001`에 바인딩 (로컬호스트 전용, 외부 네트워크에서 접근 불가)
- 설정에서 `bridgeToken`을 설정하여 Python과 Node.js 간 공유 비밀 인증 활성화
- `~/.shacs-bot/whatsapp-auth`의 인증 데이터를 안전하게 보관 (모드 0700)

### 6. 의존성 보안

**중요**: 의존성을 최신 상태로 유지하세요!

```bash
# 취약한 의존성 확인
pip install pip-audit
pip-audit

# 최신 보안 버전으로 업데이트
pip install --upgrade shacs-bot-ai
```

Node.js 의존성 (WhatsApp 브리지):
```bash
cd bridge
npm audit
npm audit fix
```

**중요 참고사항:**
- 보안 수정을 위해 `litellm`을 최신 버전으로 유지하세요
- DoS 취약점 수정을 위해 `ws`를 `>=8.17.1`로 업데이트했습니다
- `pip-audit` 또는 `npm audit`를 정기적으로 실행하세요
- shacs-bot 및 의존성에 대한 보안 권고를 구독하세요

### 7. 프로덕션 배포

프로덕션 환경에서 사용 시:

1. **환경 격리**
   ```bash
   # 컨테이너 또는 VM에서 실행
   docker run --rm -it python:3.11
   pip install shacs-bot-ai
   ```

2. **전용 사용자 사용**
   ```bash
   sudo useradd -m -s /bin/bash shacs-bot
   sudo -u shacs-bot shacs-bot gateway
   ```

3. **적절한 권한 설정**
   ```bash
   chmod 700 ~/.shacs-bot
   chmod 600 ~/.shacs-bot/config.json
   chmod 700 ~/.shacs-bot/whatsapp-auth
   ```

4. **로깅 활성화**
   ```bash
   # 로그 모니터링 설정
   tail -f ~/.shacs-bot/logs/shacs-bot.log
   ```

5. **속도 제한 사용**
   - API 제공업체에서 속도 제한 설정
   - 이상 징후에 대한 사용량 모니터링
   - LLM API에 지출 한도 설정

6. **정기 업데이트**
   ```bash
   # 매주 업데이트 확인
   pip install --upgrade shacs-bot-ai
   ```

### 8. 개발 환경 vs 프로덕션 환경

**개발 환경:**
- 별도의 API 키 사용
- 민감하지 않은 데이터로 테스트
- 상세 로깅 활성화
- 테스트용 Telegram 봇 사용

**프로덕션 환경:**
- 지출 한도가 있는 전용 API 키 사용
- 파일 시스템 접근 제한
- 감사 로깅 활성화
- 정기적인 보안 검토
- 비정상적인 활동 모니터링

### 9. 데이터 프라이버시

- **로그에 민감한 정보가 포함될 수 있습니다** - 로그 파일을 적절히 보호하세요
- **LLM 제공업체가 프롬프트를 확인합니다** - 해당 개인정보 보호 정책을 검토하세요
- **채팅 기록은 로컬에 저장됩니다** - `~/.shacs-bot` 디렉토리를 보호하세요
- **API 키는 평문으로 저장됩니다** - 프로덕션에서는 OS 키링을 사용하세요

### 10. 사고 대응

보안 침해가 의심되는 경우:

1. **즉시 침해된 API 키 폐기**
2. **비인가 접근에 대한 로그 검토**
   ```bash
   grep "Access denied" ~/.shacs-bot/logs/shacs-bot.log
   ```
3. **예상치 못한 파일 수정 확인**
4. **모든 자격 증명 교체**
5. **최신 버전으로 업데이트**
6. **관리자에게 사고 보고**

## 보안 기능

### 내장 보안 제어

✅ **입력 검증**
- 파일 작업에 대한 경로 순회 방지
- 위험한 명령어 패턴 탐지
- HTTP 요청에 대한 입력 길이 제한

✅ **인증**
- 허용 목록 기반 접근 제어 — `v0.1.4.post3` 이전에는 빈 `allowFrom`이 모든 접근을 허용했으나, `v0.1.4.post4`부터 모든 접근을 차단합니다 (`["*"]`로 명시적으로 모든 접근 허용)
- 인증 실패 시도 로깅

✅ **리소스 보호**
- 명령어 실행 타임아웃 (기본 60초)
- 출력 자르기 (10KB 제한)
- HTTP 요청 타임아웃 (10-30초)

✅ **안전한 통신**
- 모든 외부 API 호출에 HTTPS 사용
- Telegram API에 TLS 사용
- WhatsApp 브리지: 로컬호스트 전용 바인딩 + 선택적 토큰 인증

## 알려진 제한 사항

⚠️ **현재 보안 제한 사항:**

1. **속도 제한 없음** - 사용자가 무제한 메시지를 보낼 수 있음 (필요 시 직접 추가)
2. **평문 설정** - API 키가 평문으로 저장됨 (프로덕션에서는 키링 사용)
3. **세션 관리 없음** - 자동 세션 만료 없음
4. **제한된 명령어 필터링** - 명백히 위험한 패턴만 차단
5. **감사 추적 없음** - 제한된 보안 이벤트 로깅 (필요 시 강화)

## 보안 체크리스트

shacs-bot을 배포하기 전에:

- [ ] API 키가 안전하게 저장됨 (코드에 포함되지 않음)
- [ ] 설정 파일 권한이 0600으로 설정됨
- [ ] 모든 채널에 `allowFrom` 목록이 설정됨
- [ ] 루트가 아닌 사용자로 실행
- [ ] 파일 시스템 권한이 적절히 제한됨
- [ ] 의존성이 최신 보안 버전으로 업데이트됨
- [ ] 보안 이벤트에 대한 로그 모니터링
- [ ] API 제공업체에 속도 제한 설정
- [ ] 백업 및 재해 복구 계획 수립
- [ ] 커스텀 스킬/도구에 대한 보안 검토
